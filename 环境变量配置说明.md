# 环境变量配置说明

## 📋 环境变量列表

### 必需的环境变量

| 变量名 | 说明 | 示例值 | 获取方式 |
|--------|------|--------|----------|
| `VOLCANO_API_KEY` | 火山引擎API密钥 | `f9e0848d-xxxx-xxxx-xxxx-xxxxxxxxxxxx` | [火山引擎控制台](https://console.volcengine.com/ark/region:ark+cn-beijing/apiKey) |

### 可选的环境变量（有默认值）

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `VITE_TEXT_TO_IMAGE_BASE_URL` | 文生图API基础URL | `/api/volcano/api/v3/images/generations` |
| `VITE_TEXT_TO_IMAGE_MODEL` | 文生图模型 | `doubao-seedream-4-0-250828` |
| `VITE_IMAGE_TO_TEXT_BASE_URL` | 图生文API基础URL | `/api/volcano/api/v3/chat/completions` |
| `VITE_IMAGE_TO_TEXT_MODEL` | 图生文模型 | `doubao-seed-1-6` |

## 🔧 本地开发配置

### 1. 复制示例文件

```bash
cp .env.example .env
```

### 2. 填写API密钥

编辑 `.env` 文件：

```env
VOLCANO_API_KEY=你的真实API密钥
```

### 3. 启动服务

```bash
# 启动后端代理
npm run server

# 启动前端（新终端窗口）
npm run dev
```

## ☁️ Render 部署配置

### 方案1：只部署前端（静态网站）

**不需要设置环境变量**（前端通过代理访问API）

但需要先启动后端服务（Web Service）

---

### 方案2：部署前端+后端

#### 后端（Web Service）配置：

创建 Web Service，添加环境变量：

```
VOLCANO_API_KEY = 你的API密钥
VITE_TEXT_TO_IMAGE_MODEL = doubao-seedream-4-0-250828
VITE_IMAGE_TO_TEXT_MODEL = doubao-seed-1-6
PORT = 10000
```

**构建配置：**
- Build Command: `npm install`
- Start Command: `node server.cjs`

#### 前端（Static Site）配置：

**不需要环境变量**（前端通过后端代理）

**构建配置：**
- Build Command: `npm install && npm run build`
- Publish Directory: `dist`

## 🔐 安全注意事项

### ⚠️ 重要提醒

1. **永远不要**把 `.env` 文件推送到GitHub
2. **永远不要**在代码中硬编码API密钥
3. **确保** `.env` 在 `.gitignore` 中

### 检查 .gitignore

确认 `.gitignore` 包含：

```
.env
.env.local
.env.*.local
```

### 如果不小心泄露了密钥

1. 立即去火山引擎控制台删除旧密钥
2. 生成新密钥
3. 更新本地 `.env` 和Render环境变量
4. （如果已推送）删除Git历史中的敏感信息

## ❓ 常见问题

### Q1：提示"API密钥无效"？

**A：** 检查：
1. `.env` 文件中的密钥是否正确（没有多余空格）
2. 密钥是否过期或被删除
3. 重启后端服务（`npm run server`）

---

### Q2：提示"模型不存在"？

**A：** 检查：
1. 你的账号是否有权限访问该模型
2. 模型名称是否正确（区分大小写）
3. 火山引擎控制台中模型是否可用

---

### Q3：本地能用，部署后不能用？

**A：** 检查：
1. Render环境变量是否正确设置
2. 后端服务是否正常运行（Web Service）
3. 前端是否正确连接到后端URL

---

### Q4：怎么换成其他AI模型？

**A：** 修改 `.env`：

```env
# 换成其他豆包模型
VITE_TEXT_TO_IMAGE_MODEL=新模型名称
VITE_IMAGE_TO_TEXT_MODEL=新模型名称
```

重启服务即可
